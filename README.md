# PredictAll

**PredictAll** — платформа интеллектуального прогнозирования общественных настроений в социальных сетях.

Загружаешь сводку события (отчёт, новостной материал, политический документ) и формулировку вопроса — PredictAll извлекает участников и связи через GraphRAG, генерирует тысячи агентов с уникальными биографиями и взглядами, запускает многораундовую симуляцию их поведения в соцсетях и возвращает аналитический отчёт с прогнозом нарративов, инфлюенсеров-амплифаеров и точек эскалации.

Этот форк адаптирован под аудиторию Казахстана (русско-казахский билингвизм с естественным code-switching, демографический срез по районам Алматы), переведён на русский интерфейс и предназначен для симуляции общественного мнения в Threads-подобной среде.

## Архитектура

```
seed (.md/.pdf/.txt)
   │
   ▼
Zep GraphRAG ─── извлечение сущностей и связей
   │
   ▼
Persona generator ─── один агент на сущность; для демографических групп — N индивидов
   │
   ▼
OASIS engine ─── параллельная симуляция действий (post, repost, quote, like, comment)
   │
   ▼
ReportAgent ─── интервью, обзорный анализ, отчёт
```

Под капотом — [OASIS](https://github.com/camel-ai/oasis) (CAMEL-AI) для симуляционного движка и [Zep](https://www.getzep.com) для графа знаний и долговременной памяти.

## Быстрый старт

### Требования

| | |
|---|---|
| Node | 18+ |
| Python | 3.11–3.12 (поставится автоматически через `uv`) |
| `uv` | последняя |

### Конфигурация

```bash
cp .env.example .env
# отредактируй .env, заполни LLM_API_KEY и ZEP_API_KEY
```

Минимально нужны:

- `LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_NAME` — любой LLM с OpenAI-совместимым API (OpenAI, Anthropic via proxy, Qwen, и т. д.).
- `ZEP_API_KEY` — бесплатной квоты Zep Cloud достаточно для прототипирования.

Опциональные настройки региона и языка агентов задокументированы в `.env.example`:

- `SIMULATION_REGION_CONTEXT` — региональный контекст (города, имена, темы).
- `SIMULATION_AGENT_LANGUAGE` — язык контента индивидуальных агентов с правилами code-switching.
- `SIMULATION_DEMOGRAPHIC_MULTIPLIER` — сколько индивидов разворачивать из одной демографической сущности (default 5).

### Запуск

```bash
npm run setup:all   # установка зависимостей (root + frontend + backend)
npm run dev         # параллельно поднимет backend (5001) и frontend (3000)
```

Открой `http://localhost:3000`.

### Docker

```bash
docker compose -f docker-compose.deploy.yml up -d --build
```

Слушает на порту `3010`. API проксируется через тот же origin (`/api/...`).

## Пример: симуляция реакции на повышение тарифа в Алматы

В `examples/seeds/almaty_tariff_seed.md` — самодостаточный seed на ~130 строк:

- 10 публичных фигур (регулятор, аким, оппозиция, эксперты, СМИ, инфлюенсеры)
- 18 синтетических жителей по 8 районам Алматы с разными доходами, языками, возрастами и позициями
- Юридический и социальный контекст события

Загрузи этот файл в поле «Reality Seed» и попробуй запрос вроде:

> Спрогнозируй реакцию жителей Алматы на повышение тарифа на тепловую энергию в первые 72 часа. Какие нарративы появятся первыми? Кто из инфлюенсеров будет амплифицировать тему? Появятся ли в трендах призывы к митингу 22 марта?

## Что было изменено по сравнению с upstream MiroFish

- Локали: добавлены Russian и Kazakh (видимый интерфейс полностью переведён), удалена Chinese.
- Язык агентов отделён от UI-локали — `SIMULATION_AGENT_LANGUAGE` для билингвальности.
- Демографический множитель: одна абстрактная группа разворачивается в N разнообразных индивидов.
- Строгие языковые правила в ReportAgent (запрет английских вкраплений в русский отчёт).
- Шаблоны действий в Zep-памяти переведены на русский.
- Ребрендинг под локальный продукт PredictAll.
- Reddit отключён по умолчанию (фокус на Threads-подобной среде).
- Деплой-конфиг: `docker-compose.deploy.yml` для прода с независимыми портами.

Upstream остаётся доступным как `git remote upstream` для подтягивания обновлений.
